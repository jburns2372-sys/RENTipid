# Live Pilot Communication Templates

These templates must be used when communicating with participants and internal teams regarding the first real-money live payment pilot.

**CRITICAL RULES FOR ALL TEMPLATES:**
* This is a controlled live payment pilot.
* Only one low-value transaction is authorized.
* Refunds and payouts remain manual.
* Finance review is required.
* Support is available during the test.

---

## 1. Message to Pilot Renter
**Subject:** RENTipid Pilot Program: Your Upcoming Live Payment Test
**Body:**
Hi [Renter Name],
Thank you for participating in the RENTipid Phase 19B Live Payment Pilot! 
Please be aware that this is a controlled live payment pilot, and you will be charged a real, low-value payment via PayMongo. Only this single transaction is authorized for the pilot.
Please note that refunds remain manual, and Finance review is required for all transactions. Our Support team is on standby to assist you directly during this test.
Thank you,
The RENTipid Team

## 2. Message to Pilot Provider
**Subject:** RENTipid Pilot Program: Your Listing Has Been Selected
**Body:**

Your listing has been selected for our first controlled live payment test. A real renter will make a single, low-value booking.
Please be fully aware that provider payouts currently remain manual. Once the transaction clears and Finance review is completed, we will coordinate your manual payout.

Thank you,
The RENTipid Team

**Body:**

Please be prepared to perform a mandatory Finance review. Remember that refunds and payouts remain strictly manual during this phase. Support is available for escalations during the test.
Thanks,
Super Admin

## 4. Message to Super Admin

**Body:**

## 5. Message to Support/Admin
**Subject:** STANDBY: Live Pilot Execution
**Body:**
Hi Support Team,
The Phase 19B Live Payment Pilot is beginning. 
This is a controlled live payment pilot (one low-value transaction). Finance review is required, and all refunds/payouts are manual. 
Please remain on high alert for any tickets tagged "Live Payment Pilot" and respond immediately.

## 6. Message after successful payment
**Subject:** SUCCESS: Live Payment Completed
**Body:**
The Phase 19B Live Payment Pilot transaction has successfully completed! 
This single authorized low-value transaction has been securely captured. The transaction is now pending mandatory Finance review. 
Reminders: Refunds and payouts remain manual. Support remains available for any follow-up questions.

## 7. Message if payment fails
**Subject:** ALERT: Live Payment Failed
**Body:**
The Phase 19B Live Payment Pilot encountered a failure during checkout. 
This was a controlled, low-value test transaction. Since no funds were captured, manual refunds/payouts are not applicable. Support and Engineering are currently investigating.

## 8. Message if webhook is delayed
**Subject:** NOTICE: Webhook Delay Detected
**Body:**
The Phase 19B Live Payment Pilot transaction has been processed, but the webhook confirmation is delayed.
Do not panic. This is a controlled low-value transaction. Support is available. Finance review is still required, and refunds/payouts remain manual. We are monitoring the webhook logs.

## 9. Message if emergency freeze is activated
**Subject:** CRITICAL: Emergency Freeze Activated
**Body:**
The RENTipid Emergency Payment Freeze has been ACTIVATED during the Live Payment Pilot.
All checkouts are now blocked. This controlled pilot is paused. Finance review is required for any captured funds, and any necessary refunds will be processed manually. Support is actively investigating the trigger.
